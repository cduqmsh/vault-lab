
# 🔐 Vault가 뭐냐 (핵심 개념)

Vault는 한 줄로 말하면:

> **비밀번호 / API Key / DB 계정 같은 “비밀정보(Secret)”를 안전하게 저장하고 꺼내 쓰는 시스템**
> 

예:

- DB 비밀번호
- AWS Access Key
- JWT signing key
- TLS 인증서

👉 그냥 config 파일에 저장하면 위험하니까 Vault에 넣고 “필요할 때만 꺼내 씀”

---

# 🔑 1. Key shares = 5 (키를 5개로 쪼갠 것)

Vault는 root key를 그대로 주지 않고:

> 하나의 master key를 5조각으로 나눔
> 

즉:

- Key1
- Key2
- Key3
- Key4
- Key5

👉 이걸 각각 다른 사람/장소에 보관하는 구조

---

# 🔑 2. Key threshold = 3 (복구 조건)

이 뜻이 핵심입니다:

> **5개 중 3개 이상 있어야 Vault를 열 수 있음**
> 

즉:

| 상황 | 가능 여부 |
| --- | --- |
| Key 1 + Key 2 | ❌ 불가능 |
| Key 1 + Key 2 + Key 3 | ✅ 가능 |
| Key 3 + Key 4 + Key 5 | ✅ 가능 |

---

# 🔓 왜 이렇게 복잡하게 하냐?

Vault는 설계 철학이 이거예요:

> “한 사람이 모든 비밀을 알면 위험하다”
> 

그래서:

- root key 자체를 저장하지 않음
- 분산해서 관리해야만 복구 가능

---

# 🧾 Initial Root Token 의미

```
Initial Root Token
```

이건:

> Vault의 “최고 관리자 계정 비밀번호”
> 

즉:

- DB로 치면 root 계정
- AWS로 치면 root access key

⚠️ 단, 이 토큰은:

- 초기 1번만 생성됨
- 재발급 가능하지만 따로 절차 필요
- 반드시 안전하게 보관해야 함

---

# 🔁 “unseal”이 왜 필요한가?

Vault는 실행 상태가 2가지입니다:

## 1) Sealed (잠김 상태)

- Vault 켜져 있어도 접근 불가
- 데이터 암호화된 상태

## 2) Unsealed (열림 상태)

- 정상 사용 가능
- API / secret 조회 가능

---

# 🔐 그래서 흐름은 이거입니다

## 1) Vault 시작

→ sealed 상태

## 2) init 실행

→ 키 5개 + root token 생성

## 3) unseal

→ 키 3개 입력해야 열림

---

# 🧠 왜 이렇게 설계했냐 (핵심 철학)

이 구조는 이런 사고 방지 목적입니다:

- 서버 해킹됨
- 디스크 탈취됨
- 한 사람이 몰래 접근

👉 그래도 **혼자서는 Vault 못 열게 만드는 구조**

---

# 💡 쉽게 비유

Vault = 은행 금고

- Key shares = 금고 열쇠 5조각
- threshold 3 = 3명 이상이 모여야 열 수 있음
- root token = 금고 관리자 마스터 카드

---

# ⚠️ 실무에서 중요한 포인트

✔ unseal key는 절대 한 곳에 저장하지 않음

✔ 보통 3명 이상이 각각 보관

✔ 장애 복구 시에만 모아서 사용

---

# 👍 정리

당신이 본 메시지는:

> “Vault는 기본적으로 잠겨 있고, 5개의 열쇠 중 3개 이상 있어야 열 수 있으며, 최고 관리자 토큰도 따로 생성되었다”
>

**[ 설치 및 기동 ]**

Docker 기반으로 하는 것이 학습용으로는 가장 좋습니다. Ubuntu 24.04 서버 1대만 있으면 충분합니다.

제가 추천하는 목표는 단순히 "Vault 설치"가 아니라 아래 정도까지 해보는 것입니다.

```
Docker
 ├── Vault
 └── (선택) MariaDB

실습
1. Vault UI 접속
2. Secret 저장
3. User 생성
4. Policy 적용
5. Audit Log
6. AppRole
7. 스크립트에서 Secret 조회
8. (심화) Dynamic DB Credential
```

# 1. Docker 설치

확인

```
docker--version
docker compose version
```

없다면 설치

```
sudo apt update
sudo apt install docker.io docker-compose-v2-y

sudo systemctl enable--now docker
```

---

# 2. 디렉터리 생성

```
mkdir-p ~/vault/config
mkdir-p ~/vault/data
mkdir-p ~/vault/logs

cd ~/vault
```

---

# 3. Vault 설정 파일 작성

`~/vault/config/vault.hcl`

```
ui = true

disable_mlock = true

storage "file" {
  path = "/vault/data"
}

listener "tcp" {
  address = "0.0.0.0:8200"
  tls_disable = true
}
```

학습용이므로 TLS는 끕니다.

---

# 4. docker-compose.yml 작성

```
services:

  vault:
    image: hashicorp/vault:latest
    container_name: vault

    cap_add:
      - IPC_LOCK

    ports:
      -"8200:8200"

    volumes:
      - ./config:/vault/config
      - ./data:/vault/data
      - ./logs:/vault/logs

    command: vault server -config=/vault/config/vault.hcl

    restart: unless-stopped
```

실행

```
docker compose up-d
```

확인

```
dockerps
```

# 7. Unseal

컨테이너 접속

```
docker exec-it vaultsh
```

환경변수

```
exportVAULT_ADDR=http://127.0.0.1:8200
```

3번 수행

```
vault operator unseal
```

각각 위의 1-5 중 다른 키 입력

확인

```
vault status
```

결과

```
Sealed false
```
